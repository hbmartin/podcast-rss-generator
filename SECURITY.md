# Security Policy

## Supported Versions

Security fixes are provided for the latest `v2` release line.

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability. Instead, use GitHub's private vulnerability reporting for this repository, or contact the maintainer privately with enough detail to reproduce and assess the issue.

Include the affected version, a minimal reproduction when possible, and any known impact. The maintainer will acknowledge valid reports and coordinate a fix and release.
